## Conventions
### Branch
``` 
main - 배포용 브랜치 
dev - 개발용 브랜치
feat - 기능 추가
fix - 수정
``` 
``` 
브랜치 명: feat/기능명-이슈번호

- 기능명은 최대 두 단어로 작성, 각 단어는 소문자로, 구분은 하이픈
- 예시 : feat/tailcut-camera-7
``` 
### Commit 

``` 
TagName: Description #이슈번호

예시)
feat: 디자인 에셋 추가 #42
fix : 버그 수정 #25
chore : 프로젝트 세팅 #4
```
